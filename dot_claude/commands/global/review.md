Perform a thorough code review of:

$ARGUMENTS

Review criteria:
1. **Correctness**: Does the code work as intended? Any bugs or edge cases?
2. **Readability**: Is the code clear and well-documented?
3. **Best Practices**: Following language/framework conventions?
4. **Performance**: Any obvious performance issues?
5. **Security**: Any security concerns or vulnerabilities?
6. **Maintainability**: Easy to modify and extend?
7. **Testing**: Adequate test coverage?

Provide:
- Summary of findings
- Specific issues with line numbers
- Suggested improvements
- Praise for good patterns
